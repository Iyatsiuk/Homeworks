Base

  • git config --global user.name blabla - змінює ім'я
  • git config --global user.email blabal@bla - змінює email
  • git init - дозволяє проініціювати репозиторій в поточній папці
  • git status - показує поточний статус
  • git add - відслідковує зміни файлів(додає)
  • git commit - створює комміт з повідомленням
  • git log -перегляд та вивчення історіїї змін файлів проекту
  • git checkout - перемикання між гілками
  • git branch - робота з гілками в репозиторії
  • git push - заливає поточні локаньні комміти в віддалений репозиторій
  • git pull - забирає зміни з віддаленого репозиторія в локальний
  • git merge - об'єднує поточну папку з обраною
  • git clone - клонує проект з віддаленого репозиторію

.gitignore файл - за допомогою якого Git буде ігнорувати файли і директорії перелічені в цьому файлі за допомогою wildcard синтаксису

Advanced

  • git revert - відміняє дію минулих коммітів, створюючи новий, який містить всі минулі зміни
  • git reset- відміняє індексацію вказаного файлу, при цьому зберігає його вміст
  • git rebase - переміщує історію гілки поверх іншої, переписуючи історію проекту новими коммітами
  • git cherry-pick - бере зміни, внесені одним коммітом і намагається повторно використати їх у вигляді нового комміта в поточній папці
  • git stash - ховає зміни в файлах та зберігає ці зміни окремо, щоб потім можна було їх повернути
